# Orders Route

## Source

- `app/api/orders/route.ts`

## Responsibility

Creates new orders and lists company orders. During order creation it reserves an internal manual `orderNumber` from the shared `20000` sequence, skips values already used by imported WordPress orders, leaves `legacyWordpressOrderId` null, copies pending files into permanent order attachments while preserving the selected category, validates extra pickup contacts, applies saved product-card pricing snapshots with the selected price list's global settings before building summaries/items, persists native manual adjustment and hardcoded fee fields, runs alert rules from `lib/orders/alerts` for extra pickups, subcontractor price, next-day delivery, contact-customer custom time, and capacity, and skips outbound order-notification emails when the active company's `orderEmailsEnabled` flag is disabled or the order's `dontSendEmail` flag is selected. Warehouse-email copies are no longer sent from the create-order flow. The order list response also carries the derived mail-alert and notification-alert fields used by archive alert cells, maps order-creator mail attention from admin replies instead of the admin-facing inbound flag, the grouped product summary data used by the archive `Produkter` column, and saved item price lines used by the read-only PDF calculator for subcontractor and order-creator users. Archive rows are sorted by alert state first, then future delivery dates before past dates with later future dates first, then time window, then display id. Order-creator archive queries now include orders where the active membership is either the customer owner or the original creator, which makes imported WordPress orders visible without a data backfill. The archive `Store` value and store filter use the assigned `customerMembershipId` membership/user, so changing the store in the order modal updates both the table and store filtering without confusing store with customer name or label. Status filtering also normalizes legacy `fail` rows so the `failed` filter matches both old and canonical values, archive rows use normalized extra-pickup columns instead of reading `legacyWordpressRawMeta`, product summaries keep item `rawData` so custom/install descriptions remain visible, archive text search ignores letter case and spaces for names, emails, phone numbers, ids, addresses, summaries, and notes, and archive pagination now accepts up to `1000` rows for show-all date ranges.

## Functions

| Function | Description |
| --- | --- |
| `parsePositiveInt` | Parses a positive integer from query-string input with a fallback. |
| `orderMatchesSearch` | Checks an order against the normalized archive search query across searchable text, number, email, phone, assigned store, and creator fields. |
| `getTodayDateKey` | Returns today's local `YYYY-MM-DD` value for the archive delivery-date sort. |
| `hasArchiveAlert` | Checks whether a row has unresolved notification or email attention. |
| `getOrderCreatorEmailAttention` | Derives creator-facing mail attention from admin outbound replies newer than the latest creator inbound message. |
| `getArchiveEmailAttention` | Returns role-appropriate mail attention values for the archive response. |
| `getDeliveryDateGroup` | Groups delivery dates into future/today, past, and missing buckets for archive sorting. |
| `compareNullableText` | Compares nullable text fields with empty values last. |
| `compareArchiveOrders` | Sorts archive rows by alerts, delivery-date group/date, time window, and display id before pagination. |
| `POST` | Creates a new manual order with `legacyWordpressOrderId: null` and a reserved `orderNumber` from the `20000` sequence, validates that each extra pickup has either a valid phone or a valid email, applies the submitted pricing snapshot before building summaries and item rows, persists native manual adjustments plus hardcoded fee flags/minutes, transfers pending files into permanent attachments, runs the centralized alert rules, suppresses all outbound order emails when company email sending is disabled or `dontSendEmail` is selected, and no longer sends warehouse-email copies from this flow. |
| `GET` | Returns filtered, sorted, and paginated order rows for the active company, including both unread mail attention data and unresolved notification attention data for archive alert cells, along with `orderSummaryGroups`, `orderSummaryText`, distance/floor/lift fields, manual price adjustments, and `calculatorItems` built from saved item rows with product codes and prices for read-only role modals. For order creators it matches rows where the active membership is the assigned store or the stored creator so imported WordPress orders show up in the same archive, maps mail attention to admin replies instead of creator-sent messages, maps the archive store label to the assigned membership user, filters the store dropdown against `customerMembershipId`, normalizes legacy `fail` rows in both the response payload and the `failed` archive filter, reads normalized extra-pickup columns without selecting `legacyWordpressRawMeta`, applies a database prefilter for archive search to reduce the candidate set before the normalized in-memory search fallback runs, sorts matched rows by alerts, future delivery dates descending, time window, and display id before slicing the requested page, and caps `rowsPerPage` at `1000`. |
